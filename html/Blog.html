Important FAQ

What is the Document Object Model (DOM) ?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of an HTML or XML document as a tree of objects, where each object corresponds to a part of the document (like elements, attributes, or text content). With the DOM, you can programmatically access and manipulate the content, structure, and style of a web page using JavaScript.

How do you Select an element from the DOM ?

In JavaScript, you can select an element from the DOM using various methods. The most commonly used methods are:
Summary:
By ID: document.getElementById("id")
By Class: document.getElementsByClassName("class")
By Tag: document.getElementsByTagName("tag")
By CSS Selector: document.querySelector("selector") (single match) or document.querySelectorAll("selector") (multiple matches) You can use these methods to select and manipulate elements dynamically in a webpage.

What is event delegation in the context of the DOM, and why is it useful ?

Event delegation is a technique in JavaScript where instead of attaching event listeners to individual child elements, you attach a single event listener to a parent element. This parent element can handle events triggered by its children because of event bubbling, where events triggered on a child element bubble up to its parent elements in the DOM hierarchy.
How Event Delegation Works: When an event (like a click) occurs on a child element, it bubbles up through its parent elements in the DOM tree until it reaches the root (). Using event delegation, we attach a listener to a common parent element. This way, we can manage events for multiple child elements using a single event listener.

How do you manipulate an element's attributes and styles using the DOM ?

Attributes are manipulated using setAttribute(), getAttribute(), and direct property access. Styles can be changed directly using style properties or by manipulating CSS classes with classList. Using classes for styles is often cleaner and more maintainable compared to setting inline styles directly. This flexibility allows you to dynamically adjust the appearance and behavior of elements on a web page in response to user actions or other events.
